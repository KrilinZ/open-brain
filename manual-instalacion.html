INSTALLATION MANUAL

Antigravity Brain

Step-by-step guide to install and configure your AI Control Tower on macOS.

v2.0.0 · macOS arm64 (Apple Silicon)
01 — REQUIREMENTS

Before you start

Verify your machine meets these minimum requirements:

🍎

macOS

macOS 13 Ventura or later

Processor

Apple Silicon (M1/M2/M3/M4)

Storage

~200MB free disk space

Antigravity

Antigravity extension installed

ℹ️

Do you need Antigravity installed?

Yes. The app reads data directly from ~/.gemini/antigravity/. Without Antigravity, the app will open but have no data to show. Install the extension first and work with it to generate sessions and artifacts.

02 — INSTALLATION

Install antigravitybrain.dmg

1

Open the DMG

Double-click antigravitybrain.dmg. A virtual disk image with the app inside will mount.

2

Drag to Applications

Drag the Antigravity Brain icon to the Applications folder shown next to it. This copies the app to your Mac.

3

First launch (Gatekeeper)

On first launch, macOS will show a security warning because the app is unsigned. Follow these steps:

TERMINAL
# Option A: From System Settings
System Settings → Privacy & Security → "Open Anyway"

# Option B: Via Terminal (faster)
sudo xattr -rd com.apple.quarantine /Applications/Antigravity\ Brain.app
⚠️

Why does the warning appear?

The app is not signed with an Apple Developer certificate ($99/year). It's completely safe — the source code is 100% auditable. This step is only required the first time.

4

Done!

The app will automatically detect your ~/.gemini/antigravity/ directory and load all your sessions, artifacts and Knowledge Items.

✅

Autopilot mode active

On launch, Autopilot mode activates automatically, scanning sessions every 30 seconds, cleaning junk files, verifying integrity and syncing Knowledge Items.

03 — INITIAL SETUP

First steps after installing

A

Add your servers

Go to the Servers tab and click "+ Add". Enter the name, IP, SSH user and key path. The panel will be able to live-check RAM, disk, Docker and PM2 status.

EJEMPLO
# Datos de un servidor típico:
Nombre: Mi VPS Producción
IP: 123.456.789.0
SSH User: root
SSH Key: ~/.ssh/id_ed25519
B

Add your API keys

Go to the APIs tab and click "+ Add API". Enter the service name, type, key and estimated balance. Keys are stored locally on your Mac, never sent to the internet.

C

Explore your data

The Sessions, Artifacts and Knowledge Items tabs auto-populate by reading your Antigravity directory. No setup needed — just work with Antigravity and the panel updates itself.

05 — DATA STRUCTURE

Where everything lives

The app reads and writes exclusively to this directory:

FILESYSTEM
~/.gemini/antigravity/
├── brain/ ← Conversaciones y artefactos
├── knowledge/ ← Knowledge Items
├── conversations/ ← Sesiones de Antigravity
├── implicit/ ← Conversaciones implícitas
├── servers.json ← Tus servidores (se crea al añadir)
└── apis.json ← Tus API keys (se crea al añadir)
🔒

100% Local

The app sends NO data to the internet. Everything is read and written to your local disk. Your API keys, servers and sessions never leave your Mac.

06 — FAQ

FAQ

Does it work on Intel Mac?

This build is for Apple Silicon only (M1/M2/M3/M4). If you have Intel, contact me and I'll generate a compatible build.

Do I need Antigravity running at the same time?

No. The app reads files from disk. You can have Antigravity closed. But if it's running, you'll see data update in real time.

Can I easily uninstall?

Yes. Drag the app from Applications to Trash. Your Antigravity data is NOT touched — it's in ~/.gemini/antigravity/ and is independent.

Does the app modify my Antigravity data?

Only in two cases: when you click "Clean .resolved" (deletes repair files) and when Autopilot generates Knowledge Items. It never deletes sessions or artifacts.

Is it free?

Yes, 100% free. Non-profit project. Share freely.

Is the source code available?

If you want to audit the code, contact me. It's 100% transparent — you can verify there's no telemetry, tracking or data sending.

07 — TROUBLESHOOTING

If something doesn't work

🔧

"app is damaged and can't be opened"

Run in Terminal: sudo xattr -rd com.apple.quarantine /Applications/Antigravity\ Brain.app

🔧

The app opens but shows no sessions

Verify the directory exists: ls ~/.gemini/antigravity/brain/ — if empty, work with Antigravity first to generate data.

🔧

Servers won't connect via SSH

Verify the SSH key path is correct and has proper permissions: chmod 600 ~/.ssh/your_key

08 — SETUP PROMPT

Paste this in Antigravity to finish setup

Open a conversation with Antigravity and paste this prompt. It will detect your Antigravity Brain installation and help you configure servers, APIs and Knowledge Items automatically.

PROMPT READY TO COPY
I just installed Antigravity Brain (Control Tower) — the desktop app that reads my data from ~/.gemini/antigravity/ and gives me a panel with sessions, artifacts, Knowledge Items, servers and APIs. I need you to help me configure it completely: 1. **Servers**: I have these servers I want to monitor from the app. For each one I need to register: name, IP, SSH user, SSH key path, and the projects running on it. Generate the file ~/.gemini/antigravity/servers.json with the correct format. Ask me for data on each server. 2. **APIs**: I have these API keys I use in my projects. For each register: service name, type, key, estimated balance, and associated projects. Generate ~/.gemini/antigravity/apis.json. Ask me for data. 3. **Knowledge Items**: Scan my current projects and generate initial KIs in ~/.gemini/antigravity/knowledge/ so the app has context from the start. Each KI needs a metadata.json with summary and an artifacts/ with context.md. 4. **Verification**: Once files are created, open Antigravity Brain and confirm the Servers, APIs and Knowledge Items panels show data correctly. Guide me step by step. Start by asking how many servers I have and their details.
💡

What does this prompt do?

It asks Antigravity to create config files (servers.json, apis.json and Knowledge Items) on your disk. The app reads them automatically and you'll start seeing your servers, APIs and knowledge in the panel.
